기술적 도전과제

❌ 문제: Windows 배치 파일 종료

minikube.exe, kubectl.exe 실행 시 배치 스크립트가 예기치 않게 종료되는 현상

# 배치 파일에서 실행 minikube status >nul 2>&1 # ← 여기서 스크립트 종료! # 이후 코드 실행 안됨
  • call 키워드 사용 → 실패
  • PowerShell 래퍼 → 실패
  • 임시 파일 생성 → 실패
✅ 해결: PowerShell 완전 마이그레이션

모든 로직을 PowerShell로 재작성하여 근본적으로 해결

# PowerShell 스크립트 minikube start --driver=docker if ($LASTEXITCODE -ne 0) { Write-Host "[ERROR] 실패!" exit 1 } # 정상 실행됨!
  • 외부 실행 파일 호출 문제 없음
  • 강력한 에러 처리
  • Base64 디코딩 등 고급 기능

CI 파이프라인 (GitHub Actions)

name: CI/CD Pipeline on: push: branches: [master] jobs: build-and-deploy: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v3 - name: Build Docker image run: | docker build \ --build-arg GIT_COMMIT=${{ github.sha }} \ -t ghcr.io/${{ github.repository }}:${{ github.sha }} . - name: Push to GHCR run: docker push ghcr.io/${{ github.repository }} - name: Update Kustomize run: | cd deploy/k8s/overlays/prod kustomize edit set image \ app=ghcr.io/${{ github.repository }}:${{ github.sha }} git commit -m "Update to ${{ github.sha }}" git push
자동 빌드 이미지 푸시 매니페스트 업데이트 Git SHA 태깅
